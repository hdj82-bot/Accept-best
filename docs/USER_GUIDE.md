# 사용자 가이드 (베타)

academi.ai를 처음 사용하시는 분을 위한 한 바퀴 투어입니다.
가입부터 논문 검색 → 요약 → 설문 문항 생성 → 다운로드까지 평소 흐름을 따라갑니다.

> 이 문서는 **베타 사용자(연구자)** 대상입니다. 운영자/개발자용 문서는 [README](../README.md), [academi.md](../academi.md)에 있습니다.

> ℹ️ **베타 단계 안내 (2026-04 기준)**
> - 로그인은 **Google 계정만** 지원합니다. 카카오/네이버 OAuth는 베타 이후 검토 항목입니다.
> - 검색·요약·설문 생성 등 AI 작업은 현재 **동기 처리** 단계입니다(워커 큐 비활성). 요청 후 응답까지 화면을 닫지 말고 기다려주세요. 자세한 시간 가이드는 각 단계 설명을 참고하세요.
> - 자주 바뀌는 항목(가격/한도/응답시간)은 [FAQ](FAQ.md)에 모아두었습니다.

## 💡 이 도구는 답을 주지 않습니다 — 함께 묻습니다

academi.ai는 **연구자에게 되묻는 방식**으로 작동합니다. 요약·설문 문항·건강검진 등 모든 응답이 끝나면 AI가 후속 질문을 던집니다.

예시:
- 논문 요약 후 → "이 논문에서 가장 활용 가치 있는 부분이 어떤 곳인가요?"
- 설문 문항 생성 전 → "이 변수를 측정하실 때 가장 우선시하는 차원이 무엇인가요?"
- 노트 → 초안 변환 전 → "이 노트들의 공통 주장은 무엇인가요?"

**왜**:
- 연구자의 사고를 가두지 않고 확장하기 위해
- 연구자만이 알 수 있는 맥락(현장 응답자 접근성, IRB 정책, 학과 관행 등)을 답변으로 받아 결과물의 정합도를 높이기 위해
- 누적된 익명 질문-응답 패턴이 다음 사용자에게 더 좋은 질문을 만드는 토대가 됩니다 ([FAQ "AI가 자꾸 질문을 하는 이유"](FAQ.md#q-ai가-자꾸-질문을-하는데-그냥-결정해주면-안-되나요) 참고)

**답하지 않아도 됩니다**: 질문은 선택입니다. 답하면 다음 결과물이 더 정합해지고, 건너뛰어도 일반적인 응답을 받을 수 있습니다.

## 1. 가입 — Google 계정 1초 로그인

1. https://academi-ai.vercel.app 접속
2. **"Google로 시작하기"** 클릭
3. Google 계정 선택 → 권한 동의 (이메일·이름만 받습니다. 메일/드라이브 접근 권한은 요청하지 않습니다)
4. 자동으로 대시보드로 이동

## 2. 논문 검색 — 키워드 한 번으로 수집

1. 좌측 메뉴 **"논문"** → **"새 검색"**
2. 키워드 입력 (예: `transformer attention efficiency`)
3. (선택) 출처 필터: arXiv / Semantic Scholar / 둘 다
4. **검색 시작** 클릭

> ⏱️ **검색 응답 시간 (베타·동기 처리)**
> 외부 API(arXiv·Semantic Scholar) 레이트리밋이 있어 키워드/결과 수에 따라 **30초 ~ 수 분** 동안 응답이 돌아오지 않을 수 있습니다. 현재 베타는 동기 처리이므로 이 시간 동안 화면이 멈춘 것처럼 보이는 게 정상입니다. **새로고침하지 말고 기다려 주세요** — 새로고침하면 진행 중인 작업이 취소됩니다. 만약 5분 이상 응답이 없으면 페이지를 다시 열고 키워드를 좀 더 좁혀(예: 단어 1~2개 추가) 다시 시도해 보세요.

> 📸 **검색 결과 화면**
> ![search](img/search.png) _(스크린샷 슬롯)_

각 논문 카드에는 다음이 표시됩니다:

- 제목 / 저자 / 연도 / 출처
- 초록(원문)
- **AI 요약** 버튼 → 클릭 시 다음 단계로

## 3. 요약 — 한국어 요약 + 핵심 발견 + 방법론·한계

논문 카드의 **AI 요약** 버튼을 누르면 Gemini가 다음 4개 항목을 한국어 JSON 으로 반환합니다:

- **요약(summary_ko)** — 초록을 3~5문장으로 압축
- **핵심 발견(key_findings)** — 논문에서 도출한 결론·관찰 3개
- **방법론(methodology)** — 어떤 방식으로 연구했는지 1~2문장
- **한계(limitations)** — 저자가 인정한 / 추론 가능한 제약 1~2문장

동시에 임베딩(1536차원)이 저장되어 **유사 논문 추천**에도 활용됩니다.

요약은 즐겨찾기(★)할 수 있고, 컬렉션으로 묶어 둘 수 있습니다.

> ⏱️ **요약 응답 시간 (베타·동기 처리)**
> 1편당 약 **10 ~ 30초**. 현재 베타는 동기 처리라 이 시간 동안 화면 응답이 잠시 멈춥니다. 정상이며, 응답이 도착하면 자동으로 화면이 갱신됩니다. 핵심 발견은 원문 인용이 아니라 **모델이 추출·요약한 문장**이므로, 인용/연구 사용 전에는 원문 초록과 교차 확인하세요.

> 📸 **요약 화면**

## 4. 설문 문항 자동 생성 — academi.ai의 핵심 차별점

선행연구 문장을 **연구 변수에 맞게 번안된 설문 문항**으로 변환합니다.

1. 좌측 메뉴 **"설문 문항"** → **"새 생성"**
2. 베이스로 쓸 논문 선택 (검색·요약 단계에서 즐겨찾기한 논문)
3. 측정하고 싶은 **변수**(예: "교사 자율성") 입력
4. 대상(예: "한국 초등 교사 5~6학년") 입력
5. **생성 시작**

베타 단계(동기 처리)에서는 변수·논문 수에 따라 **30초 ~ 2분** 후 5~15개 문항이 만들어집니다. 응답 도착 전 화면을 닫거나 새로고침하면 작업이 취소되니, 그대로 두고 기다려 주세요. 각 문항에는 다음이 함께 보존됩니다:

| 필드 | 예시 |
|---|---|
| 원문(original_q) | "Teachers reported their perceived autonomy on a 5-point Likert scale" |
| 번안(adapted_q) | "나는 수업 운영 방식을 스스로 결정할 수 있다 (1~5점)" |
| 출처 | Smith et al. (2021), p. 14 |
| 척도 | 5점 Likert |

> 📸 **설문 문항 생성 화면**
> ![survey](img/survey.png) _(스크린샷 슬롯)_

**왜 이게 중요한가**: IRB 심사·논문 게재 시 설문 문항의 출처와 번안 근거를 요구받는 경우가 많습니다. academi.ai는 처음부터 출처를 함께 보관해 그대로 인용 가능한 형태로 제공합니다.

## 5. 다운로드 — 결과물 내보내기

생성한 결과는 다음 형식으로 다운로드할 수 있습니다.

| 영역 | 내보내기 형식 |
|---|---|
| 논문 요약 | Markdown / PDF |
| 설문 문항 | CSV (Qualtrics·Google Forms 임포트) / DOCX |
| 참고문헌 | BibTeX / RIS / APA 7판 |
| 논문 초안 | DOCX / Markdown |

각 화면 우상단의 **⬇ 다운로드** 버튼을 사용하세요.

## 6. 그 다음 단계

베타에서 이미 사용 가능한 추가 기능들:

- **논문 건강검진** — 작성 중인 초안의 구성·근거 흐름을 진단 → [건강검진 가이드](../README.md#핵심-기능)
- **연구 노트 → 초안** — 흩어진 메모를 챕터 초안으로 변환
- **버전 관리** — 자동 저장(최근 10개) + 수동 저장(무제한)
- **연구 공백 발견** — 수집한 논문 군집에서 미탐색 영역 추천

자주 묻는 질문은 [FAQ](FAQ.md)를 먼저 확인해주세요.

## 문제가 생기면

- **화면이 멈춘 것 같다**: 베타는 동기 처리라 30초~수 분 응답이 멈춰 보일 수 있습니다. **새로고침하지 말고 기다려 주세요** — 새로고침은 작업을 취소합니다. 5분 이상 응답이 없으면 페이지를 다시 열고 입력값을 좁혀 재시도하세요.
- **검색 결과가 0개**: 키워드를 영어로 바꿔보세요. arXiv·Semantic Scholar 모두 영문 색인이 우선입니다.
- **요약이 영어로 나옴**: 드물게 발생합니다. 재생성 시 한국어로 나오는 경우가 대부분입니다.
- **로그인 오류**: 베타는 Google 계정만 지원합니다. 다른 OAuth(카카오 등)는 연결되지 않습니다.
- **그 외 버그·요청**: [GitHub Issues](https://github.com/hdj82-bot/academi.ai/issues) 에 한 줄 남겨주시면 됩니다. 보안 이슈는 [README](../README.md) 하단 운영자 이메일로 비공개 제보 부탁드립니다.

베타 기간 동안 발견되는 모든 이슈는 빠르게 반영하겠습니다. 🙏
