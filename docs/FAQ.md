# 자주 묻는 질문 (FAQ)

> 사용 가이드는 [USER_GUIDE.md](USER_GUIDE.md), 운영자/개발자용은 [README](../README.md)·[academi.md](../academi.md)를 참고하세요.

---

### Q. AI가 자꾸 질문을 하는데, 그냥 결정해주면 안 되나요?

Accept.best는 **연구자가 단독으로 책임지는 결정**(주제 선정·방법론·설문 변수 등)에 대해서는 **단정적 결론을 제시하지 않습니다.** 대신 되묻는 질문을 던집니다. 이는 의도된 설계입니다.

**왜**:
1. **연구자만이 알 수 있는 맥락**이 결정에 결정적입니다 — 응답자 접근성, 학과·IRB 정책, 현장 관행, 동료 연구자와의 합의. AI가 단정하면 이 맥락을 무시한 답이 됩니다.
2. **연구자의 사고를 가두지 않기 위해.** 좋은 논문은 AI가 추천한 길이 아니라 연구자가 스스로 도달한 길에서 나옵니다.
3. **누적된 익명 답변이 시스템을 개선합니다.** 다른 연구자가 같은 변수를 측정할 때 어떤 차원을 우선했는지 (개인 식별정보 없이) 패턴화되어, 이후 더 정교한 질문을 만들어냅니다.

**그냥 결정받고 싶을 때**: 입력란에 "결정해줘" / "추천해줘"라고 명시하시면 단정형으로 응답합니다. 단, 그 답은 "일반적으로 통용되는 방식"이지 연구자의 맥락에 맞춘 답은 아닙니다.

**답하지 않아도 됩니다**: 질문은 선택이며, 건너뛰어도 작업은 진행됩니다. 다만 답한 만큼 결과물의 정합도가 올라갑니다.

---

### Q. 제 답변이 다른 사용자에게 노출되나요?

아니요. 사용자가 입력한 답변·노트·초안은 **사용자 단위로 격리**되어 다른 사용자에게 그대로 노출되지 않습니다.

단, 시스템 개선을 위해 **익명화·집계된 패턴**(예: "이 변수를 측정한 100명 중 80%가 5점 척도를 선호함")은 누적·활용됩니다. 원문 답변, 키워드, 사용자 식별 정보는 이 집계에 포함되지 않습니다. 외부 LLM(Gemini) 학습에도 사용되지 않습니다("Q. 제 논문/노트를 AI 학습에 사용하나요?" 참조).

### Q. 검색이 너무 오래 걸려요. 멈춘 건가요?

베타 단계는 **동기 처리**입니다 (Celery 워커 큐 비활성, `CELERY_DISABLED=1`). 외부 API(arXiv·Semantic Scholar) 레이트리밋과 키워드/결과 수에 따라 **30초 ~ 수 분** 응답이 멈춰 보일 수 있는데, 이는 정상입니다.

- 새로고침하지 마세요 — 새로고침은 진행 중인 작업을 취소합니다.
- 5분 이상 응답이 없으면 페이지를 다시 열고 키워드를 좁혀(예: 핵심 단어 1~2개 추가) 재시도하세요.
- 정식 출시 전에 비동기 큐(Celery)를 다시 활성화해 백그라운드 처리 + 알림으로 전환할 예정입니다.

### Q. 한국어 논문도 검색되나요?

베타 단계에서는 **arXiv**와 **Semantic Scholar** 두 출처만 사용합니다. 두 곳 모두 **영문 색인이 우선**이며 한국어 논문은 메타데이터 등록 비율이 낮아 검색 결과가 거의 안 나올 수 있습니다.

- 키워드는 영어로 입력하시는 것을 권장합니다 (예: `교사 자율성` → `teacher autonomy`).
- 한국어 학술 DB(RISS·KCI·DBpia 등) 연동은 베타 종료 후 검토 항목입니다 (해당 DB API 정책 / 라이선스 확인 필요).

---

### Q. 어떤 사람을 위한 서비스인가요?

한국에서 논문을 쓰는 **대학교 교수 / 박사과정 / 석사과정** 연구자를 1차 타깃으로 합니다. 영문 논문을 한국어로 빠르게 흡수하고, 선행연구를 기반으로 한 **설문 문항·연구 도구 제작**을 자동화하는 데 초점을 맞췄습니다.

### Q. 플랜별 한도가 어떻게 되나요?

| 플랜 | 월 논문 검색 | AI 요약 | 설문 문항 생성 | 버전 저장 | 가격 |
|---|---|---|---|---|---|
| Free | 30건 | 50건 | **불가** | 자동 10개 | 무료 |
| Basic | 200건 | 300건 | 30건 | 무제한 | 베타 기간 무료 |
| Pro | 무제한 | 무제한 | 무제한 | 무제한 | 베타 종료 후 공지 |

베타 기간 중에는 모든 기능이 열려있을 수 있으며, 정식 가격 정책은 베타 종료 직전 공지됩니다.

한도 초과 시 `QUOTA_EXCEEDED` 에러 메시지가 뜹니다. 다음 달 1일 자정(KST)에 카운터가 리셋됩니다.

### Q. 데이터는 얼마나 보관되나요?

| 항목 | 보관 |
|---|---|
| 계정 정보(이메일, 이름) | 계정 삭제 시까지 |
| 검색·요약 기록 | 12개월 (이후 자동 익명화) |
| 업로드한 초안·노트 | 계정 삭제 시까지 (사용자 요청 시 즉시 삭제) |
| 자동 저장 버전 | 최근 10개만 유지, 그 이상은 자동 삭제 |
| 수동 저장 버전 | 사용자가 직접 삭제할 때까지 |
| Sentry 에러 로그 | 30일 |

저장 위치는 Neon Postgres(Singapore region)이며, 파일 첨부는 S3에 보관됩니다.

### Q. 임베딩 모델이 OpenAI 에서 Gemini 로 바뀌었다고 들었는데, 사용자에게 영향 있나요?

내부 변경입니다. 사용자 화면·요청 방식·응답 형식은 동일하며 별도 작업이 필요 없습니다.

- 변경 전: OpenAI `text-embedding-3-small` (1536차원)
- 변경 후: Gemini `gemini-embedding-001` (output_dimensionality=1536, Matryoshka)
- 차원이 동일하므로 DB 스키마(`vector(1536)`)는 그대로입니다.

베타 이전에 수집된 일부 논문은 구 모델로 임베딩된 상태이며, 운영자가 별도 마이그레이션 스크립트로 점진 재임베딩 중입니다. 사용자가 체감하는 차이는 **유사 논문 추천 결과 미세 변화** 정도이며, 검색·요약·설문 생성 기능 동작은 동일합니다.

### Q. 로그인 / 가입 / OAuth 가 안 돼요. 카카오 / 네이버는 안 되나요?

베타 단계에서는 **Google OAuth만** 지원합니다. 카카오·네이버·이메일+비밀번호 로그인은 베타 종료 이후 검토 항목입니다.

- Google 로그인 시 받는 정보: 이메일, 이름. 그 외 권한(드라이브·메일 등)은 요청하지 않습니다.
- 회사·대학 Google Workspace 계정도 사용 가능합니다.
- 로그인 화면에서 멈추거나 무한 리다이렉트가 발생하면 브라우저 쿠키를 초기화하거나 시크릿 창에서 다시 시도해 주세요.

### Q. 제 논문/노트를 AI 학습에 사용하나요?

아니요. Accept.best는 **자체 모델 학습을 하지 않습니다.** 외부 LLM(Gemini)에 사용자 입력이 전달되는 것은 요약·설문 문항 생성 등 사용자가 직접 요청한 작업에 한정되며, Google Gemini API의 데이터 보관 정책을 따릅니다(2026-04 기준 API 입력은 모델 학습에 사용되지 않습니다).

이는 일반 챗봇 UI(Gemini, ChatGPT 웹 화면)와는 다른 정책입니다 — Accept.best는 항상 **API 경로**로만 호출합니다.

### Q. GDPR / 개인정보보호법 대응은 어떻게 되나요?

- **수집 항목**: Google OAuth로 받는 이메일, 이름. 그 외 식별정보는 받지 않습니다.
- **데이터 처리 위치**: Neon Postgres (Singapore), Render (Singapore), Vercel (글로벌 CDN, Edge는 Asia)
- **데이터 주체 권리**: 열람·정정·삭제·이동 요청은 [Issues](https://github.com/hdj82-bot/Accept-best/issues) 또는 운영자 이메일(README 하단)로 가능
- **DPO**: 1인 운영 단계라 별도 DPO는 두지 않으며, 운영자가 직접 응대합니다
- **국외 이전**: Singapore 리전 사용 사실은 가입 시 약관에 명시
- 정식 출시 시 개인정보처리방침을 별도 페이지로 공개할 예정입니다

GDPR 적용을 받는 사용자(EU 거주)께서 이용 보류를 원하시면 가입 자체를 막아드릴 수 있도록 베타 운영자에게 연락 주세요.

### Q. 계정을 삭제하고 싶어요. 데이터는 다 지워지나요?

대시보드 우상단 메뉴 → **설정 → 계정 삭제**.

삭제 시 즉시 처리되는 항목:

- 사용자 레코드 (`users` 테이블)
- 작성한 논문 초안·연구 노트·설문 문항
- 검색 히스토리
- 업로드한 파일

**보존되는 항목** (탈동기화·익명화):

- 다른 사용자에게도 영향을 주는 캐시된 논문 메타데이터(arXiv·Semantic Scholar 공개 데이터로, 사용자별이 아닙니다)
- Sentry 에러 로그(30일 후 자동 만료, 사용자 식별 정보 미포함)

삭제 후 7일 이내에는 같은 Google 계정으로 재가입 시 복구가 불가합니다. 단순 로그아웃이 필요한 경우라면 우측 상단 **로그아웃**을 사용해주세요.

### Q. 설문 문항 생성 결과의 학술적 신뢰도는 어느 정도인가요?

Accept.best는 **선행연구의 원문 문항을 그대로 보존**하고 함께 보여줍니다(원문 / 번안 / 출처 페이지). 번안된 문항은 어디까지나 **초안**이며, 다음 절차가 필요합니다:

1. 연구자가 원문과 번안문을 직접 검토
2. 필요시 전문가 안면타당도(face validity) 검토
3. 파일럿 데이터로 신뢰도(Cronbach's α 등) / 타당도 검증
4. IRB 심사 시 원문 출처와 번안 근거 함께 제출

Accept.best는 "AI가 추천하는 문항을 검토 없이 그대로 사용"하는 것이 아니라, **선행연구 탐색·번안 초안 작성 시간을 줄이는** 도구입니다.

### Q. 제가 입력한 키워드나 노트가 다른 사용자에게 노출되나요?

아니요. 모든 데이터는 **사용자 단위로 격리**됩니다 (`user_id` 칼럼 + JWT 검증). 검색으로 수집된 논문 메타데이터(arXiv·Semantic Scholar의 공개 데이터)는 캐시 효율을 위해 사용자 간 공유될 수 있지만, 이는 모두 **공개된 학술 데이터**입니다. 사용자가 작성한 노트·초안·설문 문항은 절대 공유되지 않습니다.

---

### Q. 오프라인이거나 인터넷이 느릴 때도 쓸 수 있나요?

현재는 클라우드 전용입니다. 검색·요약·설문 생성 모두 외부 API(arXiv, Semantic Scholar, Gemini) 호출이 필요해 오프라인 동작은 지원하지 않습니다. 작성한 초안은 자동 저장되니, 네트워크가 끊겨도 다시 접속 시 복구됩니다.
